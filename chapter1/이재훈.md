### JDK
- 자바 프로그래밍 언어, 자바 가상 머신, 자바 클래스 라이브러 묶음
- JDK는 자바 프로그램 개발에 필요한 최소한의 환경
- 자바 기술 시스템 전체 세대를 지칭할 때도 많이 사용

### JRE
- JDK 보다 좁은 범위
- 자바 SE API, 자바 가상 머신, 배포 기술까지 묶어 JRE라고 함
- 자바 프로그램을 실행할 수 있는 표준 환경을 제공.

### 핫스팟 VM (1인자)
- 썬*오라클 JDK와 Open JDK의 기본 가상 머신이자 가장 널리 사용되는 자바 가상 머신
- 핫 코드 감지 기술
    -  컴파일 했을 대 효과를 가장 크게 볼 수 있는 코드 영역을 런타임 시점에 알아내서 JIT 컴파일러에게 알려 준다.
    - JIT 컴파일러가 해당 코드를 메서드 단위로 컴파일한다.
    - 메서드가 자주 호출되거나 메서드 안에 시간을 많이 잡아 먹는 순환문이 있다면 JIT 컴파일을 수행해 스택을 치환한다. -> 온스택 치환
    - 컴파일러와 인터프리터가 조화롭게 협럭해 프로그램 응답 속도와 실행 성능 사이의 균형을 잡아주는 것이다.
    - 컴파일 없이 즉시 실행한 다음, 일부 코드만 백그라운드에서 컴파일 하여 치환하는 방식

### 차세대 JIT 컴파일러
- 핫스팟 가상 머신은 기본적으로 JIT 컴파일러를 두 개 내장
- 하나는 컴파일 속도가 빠른 대신 최적화를 적게 하는 클라이언트 컴파일러(C1컴파일러)
- 하나는 컴파일 속도는 느리지만 더 많은 최적화를 적용하는 서버 컴파일러(C2컴파일러)
- 인터프리터
- JDK 10 부터 하나 더 추가되었다. : 그랄 컴파일러 -> 현재는 그랄 프로젝트로 갔지만 다시 돌아올 예정
- C2를 대체하기 위해 만들어진 기술
